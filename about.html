О системе

Информация о возможностях аналитической системы и требованиях к данным

Описание системы

Аналитическая система представляет собой комплексное решение для анализа временных рядов и построения прогностических моделей. Система разработана для экономистов, финансовых аналитиков и исследователей, работающих с временными рядами в различных областях.

Импорт и обработка данных

Поддержка различных форматов данных (XLSX, CSV), автоматическая очистка и предобработка временных рядов.

Анализ стационарности

Проведение тестов на стационарность (ADF, KPSS, Phillips-Perron), определение порядка интегрированности временных рядов.

Построение моделей

Автоматический подбор параметров для различных типов моделей (ARIMA, VAR, VECM), оценка качества и статистической значимости.

Прогнозирование

Построение прогнозов на основе выбранных моделей, визуализация результатов, расчет доверительных интервалов.

Экспорт и отчеты

Формирование аналитических отчетов, экспорт результатов анализа и прогнозов в различных форматах.

Требования к данным

Для корректной работы системы и получения достоверных результатов анализа необходимо соблюдать определенные требования к формату и структуре исходных данных:

Пример структуры входного файла:
Дата ВВП Инфляция Безработица Ставка_ЦБ
01.01.2020 106.5 2.4 4.6 6.25
01.02.2020 107.2 2.3 4.5 6.25
01.03.2020 106.8 2.5 4.7 6.0
01.04.2020 103.2 3.1 5.8 5.5
... ... ... ... ...
Важно: Входной файл должен содержать колонку "Дата" в формате ДД.ММ.ГГГГ. Все остальные колонки должны содержать числовые значения для временных рядов. Имена колонок не должны содержать пробелов (можно использовать подчеркивание).

Основные требования к данным:

  • Файлы форматов XLSX или CSV
  • Наличие колонки "Дата" (обязательное требование)
  • Все временные ряды должны иметь одинаковую частоту наблюдений (месячная, квартальная, годовая)
  • Для каждого временного ряда должен быть указан непрерывный период данных без пропусков
  • Максимальный размер файла: 50 МБ

Рекомендации по подготовке данных:

  • Проверьте данные на выбросы перед загрузкой в систему
  • Убедитесь, что все временные ряды имеют общий период наблюдений
  • Для получения более точных результатов рекомендуется использовать не менее 30-50 наблюдений
  • Если в данных имеются пропуски, рекомендуется заполнить их перед загрузкой в систему

Методология анализа

Аналитическая система использует современные эконометрические методы для анализа временных рядов и построения прогностических моделей:

  • Тесты на стационарность: Расширенный тест Дики — Фуллера (ADF), Квятковского-Филлипса-Шмидта-Шина (KPSS), Филлипса-Перрона (PP)
  • Определение порядка интеграции: автоматическое определение необходимого количества разностей для достижения стационарности
  • Моделирование временных рядов: ARIMA, VAR, VECM
  • Тестирование коинтеграции: тест Йохансена, Энгла-Грейнджера
  • Анализ причинности: тест Грейнджера на причинность
  • Оценка качества моделей: информационные критерии (AIC, BIC, HQIC)

Алгоритм работы системы:

  1. Загрузка и первичная обработка данных
  2. Анализ стационарности и определение порядка интеграции временных рядов
  3. Выбор подходящего класса моделей (ARIMA, VAR, VECM) в зависимости от свойств временных рядов
  4. Автоматический подбор оптимальных параметров модели
  5. Диагностика модели и проверка адекватности
  6. Построение прогнозов и расчет доверительных интервалов
  7. Формирование аналитического отчета
